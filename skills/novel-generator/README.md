# novel-generator

女频短中篇付费小说生成器。专为**短视频投流引导付费阅读**场景设计，输出 2 万到 5 万字的中文女频小说。

## 定位

- 女频，面向投流引导付费读者
- 阅读场景：短视频引流 → 落地页前几章免费 → 章末卡点诱导充值
- 总字数 2w-5w
- 默认单章 2000-3000 字（投流落地页标准）
- 约 8-25 章
- 用户明确要求短章时，才使用 1200-1600 字
- 默认完整结局，不按无限连载处理

## 核心模型：3+3+3+1

| 维度 | 内容 |
| --- | --- |
| 4 个池子 | 爽点池（PUNCHES.md）/ 卡点池（HOOKS.md）/ 金句池（GOLDEN-LINES.md）/ **张力池（TENSION.md）** |
| 3 个锚点 | 视觉锚点 / 语言锚点 / 情绪锚点（人物专属） |
| 3 个回写 | 大纲完成后 / 第 10 章后 / 结局后 各回写一次第 1 章 |

## 适用场景

- 用户给出一句话方向，需要扩展成可投流的短中篇设定
- 用户需要女频小说大纲、章节规划、逐章正文
- 用户需要维护爽点池、卡点池、金句池、人物三锚点
- 用户需要前几章免费 + 章末强卡点的付费引流结构

## 工作流

```text
用户方向
  -> 确认题材 x 钩子手法 x 关系 x 结局
  -> 生成创作提示词 + 爽点池草表 + 金句风格示范 + 张力池草表
  -> 生成短中篇大纲（含每章爽点 / 章末卡点 / 暧昧张力 / 付费节点）
  -> 逐章创作（消费爽点池、补充新爽点、产出金句、推进暧昧张力、设置章末卡点）
  -> 更新项目状态
  -> 回写第 1 章（在大纲后、第 10 章后、结局后各一次）
  -> 完整结局
```

## 输出结构

每本小说使用独立目录，避免不同作品状态互相污染：

```text
output/<小说名>/
├── 创作提示词.md
├── 大纲.md
├── STATE.md
├── CHARACTERS.md       角色三锚点档案
├── PLOT_POINTS.md      关键事件、伏笔
├── STORY_BIBLE.md      背景设定、禁改设定
├── PUNCHES.md          爽点池
├── HOOKS.md            卡点池
├── TENSION.md          张力池（暧昧/亲密场景库）
├── GOLDEN-LINES.md     金句池
├── ERRORS.md           失败日志、第 1 章回写日志
├── 第01章_章节名.md
└── ...
```

## 快速开始

```bash
./scripts/init-novel.sh 我的小说名 --clean
```

该脚本会创建上述全部状态文件骨架，包括爽点池 / 卡点池 / 金句池 / 张力池。

然后告诉 Agent 小说方向，例如：

```text
写一篇 3 万字左右的现言女频，破镜重圆 + 追妻火葬场，
女主是离职律师，男主是她前男友。
前 5 章免费，第 5 章末做第一付费节点。
```

## 支持题材 × 钩子手法（两层结构）

题材和钩子手法可叠加。古言 + 替嫁、现言 + 重生、仙侠 + 破镜重圆都是合法组合。

### 题材

| 题材 | 重点 |
| --- | --- |
| 现言 | 职业、家庭、关系选择 |
| 古言 | 身份、礼法、权力关系 |
| 仙侠 | 宿命、牺牲、情感张力 |
| 校园 | 成长、暗恋、分别与重逢 |
| 娱乐圈 | 舆论、事业、亲密关系压力 |
| 豪门 | 阶层、利益、婚恋关系 |
| 年代 | 时代限制、家庭重负、命运挤压 |
| 悬疑爱情 | 案件推进、秘密揭露、信任建立 |

### 钩子手法

| 钩子手法 | 关键问题 |
| --- | --- |
| 破镜重圆 | 分手原因具体，复合代价可见 |
| 追妻火葬场 | 男主代价 / 跪点必须落到具体场景 |
| 久别重逢 | 重逢瞬间需要"称呼变化"或"物件触发" |
| 先婚后爱 | 婚姻是压力容器，关系升温靠共同经历 |
| 暗恋成真 | 暗恋证据链必须早早埋好 |
| 契约 / 替嫁 | 契约条款是冲突源，期限是节奏推手 |
| 重生归来 | 重生第一击 + 系统性清算节奏 |
| 穿书改命 | 原剧情压力 vs 女主自由意志 |
| 马甲 / 隐藏身份 | 至少 2-3 张马甲，分层揭露 |
| 强强 | 双方能力对等，冲突来自立场 |

## 质量标准

- 女主必须有目标、选择和成长，**普通起点 + 隐藏强项**的反差感必填。
- 男主**三件套**必填：专属感 / 反差点 / 暗恋证据链。
- 第一句必须制造信息缺口，前 300 字必须完成人设、冲突、受害/施害关系。
- 每章必须落地 1 个具体爽点（动作 / 一句话 / 一个画面）。
- 每章必须有连环锤：受虐/施压 → 女主反应/小反抗 → 情绪升级。
- 暧昧张力必须从 L1（眼神与距离）逐级递进到 L5（深度亲密），不能跳级。
- 前 6 章必须有至少 3 个 L1-L2 级暧昧场景，建立男女主身体吸引力。
- 第一付费节点适合卡在 L2-L3 暧昧中断瞬间（差一厘米的吻、被中断的触碰）。
- 每个暧昧场景必须有叙事功能，不能为擦而擦。
- 前 6 章章末必须达到"极强"或"强"卡点。
- 第一付费节点必须制造"不付费就受不了"的信息缺口。
- 第一付费节点必须卡在高潮前一秒，不能放在真相完全揭晓之后。
- 每章至少产出 1 句可截图传播的金句。
- 误会不能靠降智硬拖（主线误会最多 3-5 章）。
- 中后段必须开始收束，**最后 5 章不开新副线**。
- 结局必须解决外部问题、关系问题、女主成长问题，**并回收第 1 章的钩子和首批金句**。
- 不允许连续 3 章只甜或只虐。
