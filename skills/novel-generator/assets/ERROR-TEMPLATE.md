# 错误记录模板

用于记录质量不达标、连续性错误、状态更新遗漏和第 1 章回写日志。错误要清楚暴露，不要用含糊说法掩盖问题。

```markdown
## [NOVEL-ERR-YYYYMMDD-XXX] 问题类型

**记录时间**:
**章节**:
**严重程度**: low | medium | high | critical
**状态**: pending | resolved

### 问题

（具体写错在哪里，不写泛泛而谈）

### 影响

（影响爽点、卡点、人物、关系、伏笔、付费节点还是结局回收）

### 修正

（已经怎么改，或下一步必须怎么改）

### 避免再次发生

（更新哪份状态文件、池子或规则）
```

```markdown
## [REWRITE-CH01-YYYYMMDD-阶段] 第 1 章回写

**触发点**: 大纲完成后 / 第 10 章完成后 / 结局完成后
**状态**: pending | resolved

### 回写原因

### 修改内容

### 对后续章节的影响

### 已同步文件
- [ ] `STATE.md`
- [ ] `PLOT_POINTS.md`
- [ ] `STORY_BIBLE.md`
- [ ] `GOLDEN-LINES.md`
```

## 严重程度

| 级别 | 含义 |
| --- | --- |
| low | 局部措辞、金句不足、轻微节奏问题 |
| medium | 章节爽点弱、卡点弱、状态记录遗漏 |
| high | 人物行为断裂、伏笔冲突、付费节点失效 |
| critical | 主线崩塌、结局无法闭环、核心设定自相矛盾 |
