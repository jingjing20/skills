# 章节生成模板

用于生成女频投流付费短中篇小说章节。默认每章 2000-3000 字，必须落地 2-3 个爽点、必须章末卡点、必须产出金句、必须有男主身体引力描写，并且必须提供可切片名场面。只有用户明确要求短章时，才使用 1200-1600 字。

## 标准章节模板

```markdown
# 第XX章 [章节名]

> **本章概要**：
> **第一句话信息缺口**：
> **前 300 字任务**：（人设 + 冲突 + 谁受苦 + 谁施害 + 为什么关心）
> **本章爽点**（2-3 个，引用 PUNCHES.md 编号）：
>   - 主爽点（必须齐三件套）：动作 + 金句 + 对手/旁人当众反应特写
>   - 辅助爽点 1：
>   - 辅助爽点 2（可选）：
> **本章锤点**：[受虐/施压 → 女主反应/小反抗 → 情绪升级]
> **本章张力**（引用 TENSION.md 编号 + 级别 + 核心动作）：
> **本章身体引力**：
>   - 男主锚点触发（喉结/指节/锁骨/腰线/下颌线/眼尾/小臂/侧颈 中至少 1 个，写出具体场景）：
>   - 女主身体反应（第 4 章起每章至少 1 处：耳尖红/呼吸停/腿软/指尖蜷/颈侧热/心跳乱）：
> **情绪功能**：[心动/误会/拉扯/失控/决裂/和解/打脸/反转/追妻...]
> **关系进展**：
> **本章反派处理**（出场的反派当章必须被怼/丢脸/算回去）：
> **章末卡点**（引用 HOOKS.md 类型 + 具体落点）：
> **读者不看下一章会缺什么**：
> **可切片名场面**：
> **金句候选**（预期 1-2 句可截图传播）：

---

（正文 2000-3000 字；短章模式 1200-1600 字）

---

> **本章更新**：
> **下章目标**：
> **新增爽点 / 卡点 / 金句 / 张力**：（需同步回 PUNCHES.md / HOOKS.md / GOLDEN-LINES.md / TENSION.md）
```

## 章节检查清单

### 字数与结构

- [ ] 正文字数在 2000-3000 字；用户明确要求短章时才允许 1200-1600 字。
- [ ] 第一句话制造信息缺口，让读者产生"为什么"或"然后呢"。
- [ ] 前 300 字完成：人设、冲突、谁受苦、谁施害、读者为什么关心。
- [ ] 本章有明确场景，不是摘要堆砌。
- [ ] 开头承接上一章卡点或直接进入冲突。
- [ ] 结尾留下推进动力，章末卡点强度达标。

### 爽点与卡点

- [ ] 本章爽点数量 ≥ 2 个（1 主爽 + 1-2 辅助）；整章只有 1 个爽点直接判废。
- [ ] 主爽点三件套齐全：动作 + 金句 + 对手/旁人当众反应特写。
- [ ] 本章出场的反派当章被怼/丢脸/算回去，没有反派全身而退。
- [ ] 本章有清楚的连环锤：受虐/施压 → 女主反应/小反抗 → 情绪升级。
- [ ] 如果上一章偏虐，本章必须给中爽点（不是抽象情绪）。
- [ ] 章末卡点强度匹配章节位置（前 6 章必须达到"极强"或"强"）。
- [ ] 章末写清"不看下一章会缺什么"，不能只留氛围。
- [ ] 至少产出 1 句可截图传播的金句候选。
- [ ] 至少有 1 个可切片名场面，前 6 章尤其必须具备。

### 暧昧与身体引力

- [ ] 男主三个身体吸引力锚点（喉结/指节/锁骨/腰线/下颌线/眼尾/小臂/侧颈）至少 1 个在本章触发。
- [ ] 第 4 章起，每章至少 1 处女主身体反应（耳尖红/呼吸停/腿软/指尖蜷/颈侧热/心跳乱）。
- [ ] 暧昧张力级别符合章节窗口：前 3 章 L1-L2；第 4-6 章可 L3 卡付费；第 5-8 章 L4 释放；第 10 章后 L5 暗示式。
- [ ] L2 以上暧昧场景至少 2 处身体反应，不能只写氛围不写身体。
- [ ] L4 释放章必须有"克制中失控"动作（衬衫扣蹭开、扣住后脑、声音变哑），并伴随反思（"我以为我能控制住自己"）。
- [ ] L5 暗示场景只写清晨/痕迹/事后，不写完整生理过程。
- [ ] 暧昧场景有叙事功能（推进关系/制造卡点/触发爽点/暴露真面/付费钩子/纯引力外显）。

### 女频核心

- [ ] 女主有主动选择、行动或清晰情绪反应；亲密场景中也有主动权（攥住衣领/仰起脸/没躲开）。
- [ ] 男主三锚点（视觉/语言/情绪）至少触发 1 个。
- [ ] 关系变化有因果，不跳步。
- [ ] 甜、虐、误会、暧昧都服务人物关系。
- [ ] 男主当众强势、对女主本人克制；没有粗暴对待女主本人的描写。
- [ ] 反派或阻力有自己的逻辑链，不是降智搞笑；但当章必须有可见代价。
- [ ] 心理描写不超过本章 30%。

### 连贯性

- [ ] 与 `STATE.md` 的下一章目标一致。
- [ ] 角色状态与 `CHARACTERS.md` 一致。
- [ ] 没有违反 `STORY_BIBLE.md`。
- [ ] 已处理上一章卡点。
- [ ] 新伏笔或转折已记录到 `PLOT_POINTS.md`。
- [ ] 新爽点 / 卡点 / 金句 / 张力已同步回各自的池。

## 特殊章节模板

### 第 1 章（开局章）

```markdown
# 第01章 [章节名]

> **第一句话**：（必须是反常情境 + 被打破的预期）
> **前 300 字名场面**：（必须能直接做切片素材）
> **冲突钩**：（前 500 字内出现，必填）
> **关系钩**：（核心关系角色登场，必填）
> **反差钩**：（女主反差感的第一处暗示，必填）
> **男主线索**（必填，章末或章中必须出现）：一束没署名的花 / 一张卡片 / 一个被提到的名字 / 一个不属于女主的电话 / 监控里的背影
> **女主初始困境**：
> **谁在受苦 / 谁在施害 / 为什么关心**：
> **三章内中爽点预告**：女主当章或下一章必须有一次见血的爽点（不只是小反抗）
> **章末卡点**：（必须达到"极强"，推荐人物登场 / 称呼变化 / 信息炸弹）
> **可切片名场面**：
> **金句候选**：（必填，这句要在结局回收）

---

（正文）

---

> **回写标记**：本章需要在大纲完成、第 10 章完成、结局完成后各回写一次。
```

### 第一付费节点章（一般第 4-6 章）

```markdown
# 第XX章 [章节名]

> **付费节点标记**：第一付费节点
> **本章爽点**：（必须是大爽点 + 1 个辅助爽点，主爽点三件套齐全）
> **章末卡点强度**：极强
> **卡点类型**：（推荐 信息炸弹 / 反转预告 / 决定瞬间 / L3 暧昧中断）
> **不付费就受不了的信息缺口**：（必填，写清读者会想知道什么）
> **高潮前一秒**：（真相/证据/反击种子刚出现，但行动尚未展开）
> **暧昧张力（强烈推荐）**：L3 升级瞬间——被抵在墙上 / 差一厘米的吻 / 用指节抬起她下巴；身体反应至少 2 处
> **承诺爽感**：（下章必须立刻兑现）
> **禁止提前满足**：（本章不能把哪些答案讲完）

---

（正文）

---

> **付费转化关键点**：本章末必须让读者产生"必须付费看下一章"的冲动。
> **下章兑现要求**：下一章开头必须承接本章卡点，并在 800 字内兑现至少一个爽感。
```

### L4 释放章（一般第 5-8 章，男主第一次失控）

```markdown
# 第XX章 [章节名]

> **L4 释放标记**：男主第一次失控
> **触发情境**：（必须有情绪重量——误会爆发 / 长时间蓄能 / 离别前 / 她在他面前哭了）
> **失控动作**：（扣住后脑 / 掐住下颌 / 抵在墙上 / 抓住手腕扯回来）
> **衣物细节**：（衬衫第 X 颗扣子蹭开 / 领带松开 / 西装外套撩起）
> **女主身体反应**（至少 3 处）：（推/没推动/攥住衣领/腿软/缺氧/被咬到的下唇）
> **吻后反思**（必填）：（"我以为我能控制住自己"——男主必须承认自己破功了）
> **本章爽点**：
> **章末卡点**：

---

（正文）

---

> **关系变化标记**：L4 释放后两人关系再回不去；中后段可继续 L3-L4 拉扯，但不能装作没发生过。
```
