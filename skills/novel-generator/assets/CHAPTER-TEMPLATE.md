# 章节生成模板

用于生成女频短中篇小说章节。默认每章 2000-3000 字。最高优先级是真人语感，其次才是吸引力、爽点和卡点。模板字段只做后台约束，正文不能写出填表味。

## 标准章节模板

```markdown
# 第XX章 [章节名]

> **本章概要**：
> **第一句话信息缺口**：
> **前 300 字任务**：（进入冲突 / 关系压力 / 反常任务）
> **本章核心动作**：（女主做了什么选择或反击）
> **本章关系推进**：
> **本章吸引力**：（爽点 / 欲感 / 反转 / 旧物 / 公开站队，选最重要的 1-2 个）
> **关系进展**：
> **章末卡点**（引用 HOOKS.md 类型 + 具体落点）：
> **读者不看下一章会缺什么**：
> **金句候选**（预期 1-2 句可截图传播）：
> **真人语感要求**：
>   - 本章主要口吻：（冷静克制 / 口语互刺 / 情绪爆发 / 酸涩拉扯 / 轻松暧昧）
>   - 本章禁用 AI 腔：（连续比喻 / 抽象总结 / 模板身体反应 / 广告词金句 / 设定解释台词）
>   - 本章对白检查：（至少 5 句对白必须像真人在该情绪下会说的话）

---

（正文 2000-3000 字；短章模式 1200-1600 字）

---

> **本章更新**：
> **下章目标**：
> **新增爽点 / 卡点 / 金句 / 张力**：（需同步回 PUNCHES.md / HOOKS.md / GOLDEN-LINES.md / TENSION.md）
```

## 章节检查清单

### 结构

- [ ] 正文字数在 2000-3000 字；用户明确要求短章时才允许 1200-1600 字。
- [ ] 第一句话制造信息缺口，让读者产生"为什么"或"然后呢"。
- [ ] 前 300 字进入冲突、关系压力或反常任务。
- [ ] 本章有明确场景，不是摘要堆砌。
- [ ] 开头承接上一章卡点或直接进入冲突。
- [ ] 结尾留下下一章必须看的问题。

### 吸引力

- [ ] 本章至少有一个明确吸引力来源：反击、欲感、反转、旧物、公开偏爱、身份揭露或关系失控。
- [ ] 关键胜利由女主主动选择、布局、留证或反杀触发；男主只能加码。
- [ ] 如果上一章偏虐，本章给读者情绪补偿或反抗信号。
- [ ] 如果上一章是付费节点，本章开头先兑现，再进入新铺垫。
- [ ] 章末写清"不看下一章会缺什么"，不能只留氛围。

### 关系与欲感

- [ ] 亲密或欲感场景改变关系、暴露人物或制造卡点。
- [ ] 身体反应不是机械轮换，必须对应当下权力变化和人物选择。
- [ ] 男主可以强势，但不能无视女主意愿；张力来自拉扯、克制和破功。

### 女频核心

- [ ] 女主有主动选择、行动或清晰情绪反应；亲密场景中也有主动权（攥住衣领/仰起脸/没躲开）。
- [ ] 男主三锚点（视觉/语言/情绪）至少触发 1 个。
- [ ] 关系变化有因果，不跳步。
- [ ] 甜、虐、误会、暧昧都服务人物关系。
- [ ] 反派或阻力有自己的逻辑链，不靠降智搞笑。
- [ ] 心理描写不超过本章 30%。

### 真人语感

- [ ] 已按 `references/prose-style-guide.md` 做一轮去 AI 腔重写。
- [ ] 正文没有复述“爽点、卡点、张力、四件套”等后台字段。
- [ ] 抽象情绪判断已压到最低，关键情绪主要由动作、对白、物件承载。
- [ ] 明显比喻不过密；没有连续两段都靠比喻推进。
- [ ] 对白有停顿、打断、反问或潜台词，不像角色在替作者解释设定。
- [ ] 同一章没有机械重复“耳尖红、呼吸乱、腿软、指尖蜷”等身体反应。
- [ ] 至少删掉 3 句漂亮但不推进冲突、关系或卡点的句子。
- [ ] 随机抽查 5 句对白，真人在当前情绪和关系里能说出口。

### 连贯性

- [ ] 与 `STATE.md` 的下一章目标一致。
- [ ] 角色状态与 `CHARACTERS.md` 一致。
- [ ] 没有违反 `STORY_BIBLE.md`。
- [ ] 已处理上一章卡点。
- [ ] 新伏笔或转折已记录到 `PLOT_POINTS.md`。
- [ ] 新爽点 / 卡点 / 金句 / 张力已同步回各自的池。

## 特殊章节模板

### 第 1 章（开局章）

```markdown
# 第01章 [章节名]

> **第一句话**：（必须是反常情境 + 被打破的预期）
> **前 300 字名场面**：（必须能直接做切片素材）
> **冲突钩**：（前 500 字内出现，必填）
> **关系钩**：（核心关系角色登场，必填）
> **反差钩**：（女主反差感的第一处暗示，必填）
> **男主线索**（必填，章末或章中必须出现）：一束没署名的花 / 一张卡片 / 一个被提到的名字 / 一个不属于女主的电话 / 监控里的背影
> **女主初始困境**：
> **谁在受苦 / 谁在施害 / 为什么关心**：
> **三章内中爽点预告**：女主当章或下一章必须有一次见血的爽点（不只是小反抗）
> **章末卡点**：（必须达到"极强"，推荐人物登场 / 称呼变化 / 信息炸弹）
> **可切片名场面**：
> **金句候选**：（必填，这句要在结局回收）

---

（正文）

---

> **回写标记**：本章需要在大纲完成、第 10 章完成、结局完成后各回写一次。
```

### 第一付费节点章（一般第 4-6 章）

```markdown
# 第XX章 [章节名]

> **付费节点标记**：第一付费节点
> **本章兑现**：（本章给了读者什么情绪补偿）
> **卡点类型**：（信息炸弹 / 反转预告 / 决定瞬间 / 欲感中断）
> **不付费就受不了的信息缺口**：（必填，写清读者会想知道什么）
> **高潮前一秒**：（真相/证据/反击种子刚出现，但行动尚未展开）
> **承诺兑现**：（下一章开头立刻给什么）
> **禁止提前满足**：（本章不能把哪些答案讲完）

---

（正文）

---

> **付费转化关键点**：本章末必须让读者产生"必须付费看下一章"的冲动。
> **下章兑现要求**：下一章开头必须承接本章卡点，先兑现，再铺新信息。
```
