# 女频短中篇付费小说生成器

用于根据用户方向，规划并创作面向短视频投流引导付费阅读场景的女频短中篇小说。目标不是慢热文学短篇，而是完整闭环、爽点密集、章末卡点强、前几章能把读者钩到付费的商业文本。

投流模式下，**转化优先级高于文学性**。文采只能服务信息缺口、情绪爆点、付费卡点和可切片名场面；如果一句漂亮话没有制造疑问、冲突或传播点，删掉。

## 默认强吸引力模式

本 skill 默认进入**强吸引力 / 强转化 / 强欲感**档位。目标不是"好看"，而是让读者产生三种即时反应：想看反派马上付出代价、想看男女主马上越界、想看下一章马上兑现。

执行优先级：

1. **刺激强度 > 文学质感**：慢慢铺、轻轻虐、淡淡甜，都要让位给强冲突、强反扑、强卡点。
2. **可见代价 > 情绪描述**：反派不能只脸色难看，必须失去面子、钱、资格、盟友、证据主动权、圈层位置中的至少一项。
3. **女主掌控 > 男主救场**：男主可以偏爱和加码，但关键胜利必须由女主提前布局、主动选择或当场反杀触发。
4. **欲感外显 > 氛围含蓄**：男女主身体吸引力、触碰、呼吸、衣物细节、吻后痕迹可以直给；过程留白，但结果和身体反应要写得清楚。
5. **付费后立刻爽 > 延迟满足**：第一付费节点下一章前 800 字必须兑现一个中爽以上，不允许先铺氛围、回忆、赶路、换衣服。

## 硬性定位

| 项目 | 默认规则 |
| --- | --- |
| 读者定位 | 女频，投流引导付费读者 |
| 阅读场景 | 短视频引流 -> 落地页前几章免费 -> 章末卡点诱导充值 |
| 总字数 | 2 万 - 5 万字 |
| 单章字数 | 默认 2000 - 3000 字；用户明确要求短章时才压到 1200 - 1600 字 |
| 章节规模 | 约 8 - 25 章，按总字数倒推 |
| 默认免费章 | 前 3 - 5 章 |
| 默认输出 | `output/<小说名>/` 下的提示词、大纲、章节和状态文件 |

不要按长篇网文处理。不要设计超长大地图、无限扩张体系、多阶段终局。核心是：第一句给疑问，前 300 字给冲突，第 1 章末必须有男主线索，前 3 章给中爽（不是小爽），每章至少 2-3 个爽点，免费章末让读者不得不买。

## 投流铁律

1. **第一句话必须制造信息缺口**：反常情境 + 被打破的预期，让读者问“为什么”或“然后呢”。
2. **前 300 字必须完成代入**：谁受苦、谁施害、冲突是什么、女主为什么值得关心，必须用场景和行动展示。
3. **第 1 章末必须有男主线索**：不要求男主当章露脸，但必须有他的痕迹——一束没署名的花、一张卡片、一个被提到的名字、一个不属于女主的电话、一段监控里的背影。读者第一章就要知道有人在看着她。
4. **三章内必须有中爽点**：不是抽象情绪，必须落到具体动作 + 金句 + 对手或旁人当众反应特写 + 可见代价。证据留存、退路铺好这种"小爽"只能作辅助，主爽点要见血。
5. **每章爽点密度 2-3 个**：1 个主爽点（动作+金句+反应+代价四件套齐全）+ 1-2 个辅助爽点（伏笔触发、旁人替读者发声、男主小动作）。整章只有受虐没有任何爽感的写法直接废掉。
6. **每章必须有连环锤结构**：受虐 / 施压 -> 女主反应或反抗 -> 情绪升级 -> 章末卡点。
7. **每章必须有身体引力描写**：男主三个身体吸引力锚点（喉结/指节/锁骨/腰线/下颌线/眼尾/小臂/侧颈 选 3）至少 1 个在本章触发；从第 3 章起每章至少 1 处女主身体反应（耳尖红、呼吸乱、腿软、颈侧热、心跳乱、指尖蜷、喉咙发干、唇齿微开）。
8. **付费点不是高潮，是高潮前一秒**：真相、证据、反击种子刚出现，具体行动必须留到付费章兑现；付费后下一章前 800 字内必须兑现一个中爽以上。
9. **羞辱式名场面优先**：当众羞辱、真相揭露、身份反转、公开站队、下跪代价、离开决定，对手必须有当众失态/愣住/脸色苍白/手抖/咬破嘴唇的特写。没有当众反应的爽点都是哑炮。

## 爽点强度分级

每个爽点必须标注强度，不标注不允许进入大纲或章节正文。

| 等级 | 定义 | 最低兑现 |
| --- | --- | --- |
| S1 小爽 | 轻度反扑、旁人注意、男主偏爱小动作 | 对手失态 / 旁人惊讶 / 女主拿回一点主动权 |
| S2 中爽 | 完整打脸、公开偏爱、局部身份揭露 | 对手当众丢脸 + 损失资源、机会、盟友或舆论主动权 |
| S3 大爽 | 身份反转、旧账清算、公开站队、追妻代价 | 对手被踢出局 / 跪求 / 破产入狱风险 / 社会性死亡；关系权力彻底倒转 |

前 6 章强度要求：第 1 章至少 S1；第 2 章至少 S1；第 3 章必须 S2；第一付费节点必须 S2-S3；付费后第一章必须在前 800 字兑现 S2 以上；第 6 章必须继续 S2，不允许降温。

## 主爽点四件套

主爽点必须齐四件，缺一件就降级为辅助爽点：

1. **动作**：一个可视化动作，能被短视频切成画面。
2. **金句**：一句可截图、可复述、能压住对手的台词或心理独白。
3. **当众反应**：对手或旁人当场失态、愣住、脸色苍白、手抖、咬破嘴唇、被自己人抛弃。
4. **可见代价**：对手必须失去至少一样东西：脸面、钱、资格、合同、盟友、名声、证据主动权、家族/公司位置。

只有"脸色变了"不算强爽；只有男主护着女主也不算强爽。最强写法是：女主提前设局 -> 对手主动踩坑 -> 男主公开加码 -> 对手当场失去东西。

## 投流尺度规则

| 维度 | 上限 | 下限 |
| --- | --- | --- |
| 爽点戾气 | 破产、入狱、社会性死亡、当众跪求、被踢出圈层 | 当众失态、丢脸、被嘲讽 |
| 反派结局 | 入狱、破产、被流放出局；不写家破人亡、终生残疾、死亡 | 至少要付出可见代价，不允许全身而退 |
| 霸总占有欲 | 当众揽腰、宣示主权、撕合同、拽人离场、把人按在墙上对峙、把外套盖在她身上 | 不能粗暴对待女主本人、不能无视女主意愿、不能伤害女主 |
| 软色情身体反应 | 耳尖红、呼吸乱、腿软、颈侧热、心跳乱、指尖蜷、喉咙发干、唇齿微开、缺氧、被压时身体轻微挣动、推不开后攥紧衣领 | 不写完整生理过程，不写器官，不写到平台会下架的程度 |
| 软色情暗示式细节 | 衬衫扣半开、领带松到第二颗、被咬过的下唇、清晨身上不属于自己的衣物、床单褶皱、手腕红痕、锁骨那块淤红、腰侧指印、被压在身下的角度、他用拇指抹去她唇角的湿意、"昨晚没收住" | 不写器官、不写动作具体过程 |
| 打脸 | 对手当场愣住、失态、当众丢脸、被自己人抛弃、跪求 | 对手必须有自己的逻辑链，不靠降智搞笑 |

## 默认选择

用户未指定题材时，按投流转化优先级选：

| 优先级 | 赛道 | 原因 |
| --- | --- | --- |
| 1 | 现言追妻火葬场 | 背叛、轻视、后悔、代价最容易快速共鸣 |
| 2 | 重生 / 穿越复仇爽文 | 爽感直接，反转周期短，适合密集打脸 |
| 3 | 替身 / 契约婚姻觉醒文 | 代入强，情绪标签清晰，容易做离开和追悔卡点 |

如果用户只说“写个女频小说”，默认：现言 + 追妻火葬场 + 双重背叛 + 绝地反击，约 3 万字，12-16 章，前 4 章免费，第 4 章末第一付费节点，HE。

题材和钩子手法是两层：古言 + 替嫁、现言 + 重生、仙侠 + 破镜重圆都合法。不要把它们当互斥项。

## 核心资产

| 资产 | 用途 |
| --- | --- |
| `PUNCHES.md` | 爽点池：打脸、反转、追妻代价、身份揭露等，逐章消费和新增 |
| `HOOKS.md` | 卡点池：人物登场、信息炸弹、动作未完、反转预告等 |
| `GOLDEN-LINES.md` | 金句池：可截图、可投放、可回收的句子 |
| `TENSION.md` | 张力池：暧昧氛围、身体语言、身体吸引力锚点、亲密递进；按 L1-L5 分级 |

每个核心角色必须有三锚点：视觉锚点、语言锚点、情绪锚点。男主额外必须有身体吸引力锚点（喉结/指节/锁骨/腰线/下颌线/眼尾/小臂/侧颈/小肌肉 选 3），每个反复触发，全篇至少出现 3 次。不写满锚点的角色，不允许参与章节生成。

## 工作流

```text
用户方向
  -> 确认关键约束
  -> 创作提示词 + 四个池子草表
  -> 短中篇大纲
  -> 逐章生成
  -> 状态更新与质量检查
  -> 回写第 1 章
  -> 完整结局
```

## 第一步：确认关键约束

用户信息不足时，问最少的问题。不要一次问十个问题。

必须确认或自行给出默认值：

1. 题材：现言、古言、仙侠、年代、悬疑、豪门、娱乐圈、校园等。
2. 钩子手法：先婚后爱、破镜重圆、久别重逢、暗恋成真、契约/替嫁、追妻火葬场、重生归来、穿书改命、马甲/隐藏身份、强强等。
3. 篇幅：2 万、3 万、5 万，或对应章节数。
4. 女主设定：身份、欲望、伤口、成长目标、反差感。
5. 男主/关键对手设定：身份、关系张力、阻力来源、反差点。
6. 结局倾向：HE、BE、开放式，默认 HE。

提示词补全规则见 `references/prompt-guide.md`。

## 第二步：生成创作提示词

保存到 `output/<小说名>/创作提示词.md`，使用 `assets/PROMPT-TEMPLATE.md`。

提示词必须包含：

- 基本定位：题材、钩子手法、总字数、章节数、免费章数、第一付费节点、结局。
- 核心卖点：一句话梗概、第一句话信息缺口、前 300 字名场面、第一付费点买点。
- 女主：反差感、外在目标、内在伤口、成长弧、三锚点。
- 男主：专属感、反差点、暗恋证据链、三锚点。
- 四个池子：爽点池、卡点池、金句池、张力池。

## 第三步：生成短中篇大纲

保存到 `output/<小说名>/大纲.md`，使用 `assets/OUTLINE-TEMPLATE.md`。结构参考见 `references/plot-structures.md`。

大纲必须做到：

- 控制在 2 万 - 5 万字可完成的主冲突内。
- 第 1 付费节点放在第 3-6 章之间。
- 第一付费节点卡在高潮前一秒，下一章必须兑现爽感。
- 每章规划核心事件、锤点、爽点/反抗、张力、章末卡点、付费理由、可切片名场面。
- 最后 5 章不引入新副线、新反派、新身份反转。

## 第四步：建立项目状态

不要把用户小说状态写进 skill 目录本身。每部小说使用独立工作区：

```text
output/<小说名>/
├── 创作提示词.md
├── 大纲.md
├── STATE.md
├── CHARACTERS.md
├── PLOT_POINTS.md
├── STORY_BIBLE.md
├── PUNCHES.md
├── HOOKS.md
├── TENSION.md
├── GOLDEN-LINES.md
├── ERRORS.md
└── 第XX章_章节名.md
```

`STATE.md` 使用 `assets/STATE-TEMPLATE.md`。每次写新章节前必须读取状态文件和四个池子，不允许凭记忆续写。

## 第五步：逐章生成

每章保存为 `output/<小说名>/第XX章_章节名.md`，使用 `assets/CHAPTER-TEMPLATE.md`。

章节必须满足：

| 要素 | 要求 |
| --- | --- |
| 字数 | 默认 2000-3000 字；短章模式 1200-1600 字 |
| 事件 | 至少推进一个真实事件 |
| 开头 | 第一句必须有信息缺口；普通承接章也不能纯氛围开头 |
| 前 300 字 | 交代谁受苦、谁施害、冲突是什么、读者为什么关心 |
| 爽点 | 落地 2-3 个爽点（1 主爽 + 1-2 辅助），主爽点必须有动作+金句+对手或旁人当众反应特写 |
| 反派处理 | 本章出场的反派必须当章被怼/丢脸/被算回去，不允许全身而退 |
| 锤点 | 有“施压/受虐 -> 反应/反抗 -> 情绪升级”的链条 |
| 张力 | 按 L1-L5 递进，不跳级；详见 `references/chemistry-guide.md` |
| 身体引力 | 男主三个身体吸引力锚点至少 1 个本章触发；第 3 章起每章至少 1 处女主身体反应；L2 以上至少 2 处身体反应 |
| 女主 | 必须有选择、行动、布局或认知变化；关键胜利不能只靠男主救场 |
| 章末 | 必须有强卡点，前 6 章强度更高 |
| 金句 | 至少产出 1 句可截图传播的金句 |
| 名场面 | 前 6 章每章至少 1 个可切片画面 |

## 第六步：状态更新与质量检查

每章生成后自检。关键项失败时，先修正本章，再进入下一章。

必须同步更新：

- `STATE.md`：当前进度、上一章结尾、下一章目标、待处理事项。
- `CHARACTERS.md`：新角色、关系变化、三锚点触发。
- `PLOT_POINTS.md`：关键事件、伏笔、未解决问题。
- `PUNCHES.md` / `HOOKS.md` / `GOLDEN-LINES.md` / `TENSION.md`：新增和已消费项。
- `ERRORS.md`：质量不达标、连续性错误、第 1 章回写日志。

质量检查详见 `assets/CHAPTER-TEMPLATE.md` 和 `references/failure-patterns.md`。

## 第七步：回写第 1 章

短中篇第 1 章必须回写 3 次：

| 触发点 | 回写目的 |
| --- | --- |
| 大纲完成后 | 第 1 章三重钩对应大纲核心冲突 |
| 第 10 章完成后 | 第 1 章伏笔和锚点匹配已写章节 |
| 结局完成后 | 第 1 章埋下结局呼应：金句、动作、物件 |

每次回写必须在 `ERRORS.md` 记录“第 1 章回写日志”，写明改了什么、为什么改。

## 图解规则

默认使用 Mermaid，不主动调用图像生成工具。只有用户明确要求生成图片资产时，才使用图像生成。

适合图解：人物关系图、情绪曲线、爽点/卡点/金句/张力分布图、章节结构图。

## 禁止事项

- 不要默认写成男频升级流。
- 不要使用无限升级、无限地图、无限反派递进。
- 不要让女主成为被拯救的道具。
- 不要为了虐而虐；每个虐点必须服务关系或人物。
- 不要靠角色降智制造冲突；但对手可以被算计、被反将、当场失态。
- 不要写慢热文艺短篇；前 6 章必须有爽点和卡点。
- 不要用纯文学氛围句开篇，除非同一句同时制造信息缺口。
- 不要连续两章只有受虐，没有小反抗、留证、退路或旁人替读者发声。
- 不要把付费点放在真相完全揭晓之后。
- 不要把不同小说的状态混写到同一个共享状态目录。
- 不要跳过大纲直接写正文，除非用户明确要求即兴创作。
- 不要在最后 5 章引入新副线、新反派、新身份反转。
- 不要让章末“什么都没发生”。
- 不要让爽点章里反派全身而退；每个出场反派当章必须有可见代价。
- 不要跳过暧昧递进级别；L4 不能在前 5 章，L5（含暗示式清晨/痕迹）不能在前 10 章。
- 不要写到平台会下架的完整生理过程；身体反应和暗示式生理细节是允许的，详见“投流尺度规则”。
- 不要让暧昧场景没有叙事功能；窒息感本身就是合法功能。
- 不要让男主完全失控（粗暴、无视女主意愿、伤害女主）；克制中短暂失控是合法的暧昧高峰。
- 不要写"高级感"克制爽文；投流读者要的是直给的爽，不是含蓄的爽。
