# 女频短中篇付费小说生成器

用于根据用户给出的方向，规划并创作 **面向短视频投流引导付费阅读场景** 的女频短中篇小说。默认目标是写出一部完整闭环、爽点密度高、章末卡点强、能在前几章把读者钩到付费的作品，而不是慢热文学短篇。

投流模式下，**转化优先级高于文学性**。文采只能服务信息缺口、情绪爆点、付费卡点和可切片名场面；如果一句漂亮话没有制造疑问、冲突或传播点，删掉。

## 硬性定位

| 项目 | 默认规则 |
| --- | --- |
| 读者定位 | 女频，投流引导付费读者 |
| 阅读场景 | 短视频引流 → 落地页前几章免费 → 章末卡点诱导充值 |
| 总字数 | 2 万 - 5 万字 |
| 单章字数 | 默认 2000 - 3000 字（投流落地页标准）；用户明确要求短章时才压到 1200 - 1600 字 |
| 章节规模 | 约 8 - 25 章，按总字数倒推 |
| 默认免费章 | 前 3 - 5 章 |
| 核心目标 | 三秒钩住 + 前 300 字代入 + 连环锤情绪 + 付费前一秒卡点 + 完整结局 |
| 默认输出 | `output/<小说名>/` 下的提示词、大纲、章节和状态文件 |

不要按长篇网文处理。不要写慢热文艺短篇。不要设计超长大地图、无限扩张体系、多阶段终局。**投流付费短篇的核心是：第一句给疑问，前 300 字给冲突，前 3 章给小爽，免费章末让读者不得不买。**

## 投流铁律

1. **第一句话必须制造信息缺口**：反常情境 + 被打破的预期，让读者问"为什么"或"然后呢"。
2. **前 300 字必须完成代入**：谁受苦、谁施害、冲突是什么、女主为什么值得关心，必须用场景和行动展示。
3. **开篇第一段必须出金句或画面**：要么能截图传播，要么能直接剪成 15 秒短视频素材。
4. **三章内必须有小爽点**：证据被保存、退路被铺好、女主第一次反击、旁人替读者发声，任选其一，但不能让女主连续挨打。
5. **每章必须有连环锤结构**：受虐 / 施压 → 女主反应或反抗 → 情绪升级 → 章末卡点。
6. **付费点不是高潮，是高潮前一秒**：真相、证据、反击种子刚出现，具体行动必须留到付费章兑现。
7. **名场面优先**：当众羞辱、真相揭露、身份反转、公开站队、下跪代价、离开决定，都要能独立切片投放。

## 默认赛道优先级

用户没有指定题材时，按投流转化优先级选择：

| 优先级 | 赛道 | 适用原因 |
| --- | --- | --- |
| 1 | 现言追妻火葬场 | 用户基数最大，背叛、轻视、后悔、代价最容易快速共鸣 |
| 2 | 重生 / 穿越复仇爽文 | 爽感直接，反转周期短，适合密集打脸 |
| 3 | 替身 / 契约婚姻觉醒文 | 代入强，情绪标签清晰，容易做离开和追悔卡点 |

其他赛道可以写，但必须证明它能在前 300 字制造强冲突，并在前 6 章完成钩住、代入、共鸣、期待、付费。

## 核心模型：3+3+3

| 维度 | 内容 |
| --- | --- |
| 3 个池子 | 爽点池 / 卡点池 / 金句池 |
| 3 个锚点 | 视觉锚点 / 语言锚点 / 情绪锚点（人物专属） |
| 3 个回写 | 大纲完成后回改第1章 / 第10章后再回改第1章 / 结局完成后回改第1章 |

爽点池、卡点池、金句池是这套 skill 的核心资产。它们在写大纲时建立，逐章生成时消费和补充，最终决定作品能不能让读者付费。

## 快速参考

| 场景 | 操作 |
| --- | --- |
| 用户只给一个方向 | 先补齐篇幅、题材、钩子手法、女主、男主/核心关系、结局倾向 |
| 用户要开始写 | 先生成 `创作提示词.md`、`大纲.md`、`PUNCHES.md`（爽点池）、`HOOKS.md`（卡点池），请用户确认 |
| 用户要继续下一章 | 先读 `STATE.md`、`CHARACTERS.md`、`PLOT_POINTS.md`、`STORY_BIBLE.md`、`PUNCHES.md`、`HOOKS.md` |
| 出现新角色/关系变化 | 更新 `CHARACTERS.md` |
| 发生关键转折/伏笔 | 更新 `PLOT_POINTS.md` |
| 章节产出爽点或新钩 | 更新 `PUNCHES.md` / `HOOKS.md` |
| 章节产出可截图金句 | 更新 `GOLDEN-LINES.md` |
| 章节质量不达标 | 记录到 `ERRORS.md`，修正后再继续 |
| 写完大纲 / 第10章 / 结局 | 触发回写第1章检查 |

## 工作流

```
用户方向
  ↓
确认关键约束（题材 × 钩子手法 × 关系 × 结局）
  ↓
创作提示词 + 爽点池草表 + 金句风格示范
  ↓
短中篇大纲（含每章爽点 / 章末卡点 / 付费节点）
  ↓
逐章生成（消费爽点池、补充新爽点、产出金句、设置章末卡点）
  ↓
状态更新与质量检查
  ↓
回写第1章（在大纲后、第10章后、结局后各一次）
  ↓
完整结局
```

## 第一步：确认关键约束

用户信息不足时，先问最少的问题。不要一次问十个问题。

必须确认或自行给出默认值：

1. **题材**：现言、古言、仙侠、年代、悬疑、豪门、娱乐圈、校园等。
2. **钩子手法**（可叠加题材）：先婚后爱、破镜重圆、久别重逢、暗恋成真、契约/替嫁、追妻火葬场、重生归来、穿书改命、马甲/隐藏身份、强强等。
3. **篇幅**：2 万、3 万、5 万，或对应章节数。
4. **女主设定**：身份、欲望、伤口、成长目标、反差感（外表与内核错位）。
5. **男主/关键对手设定**：身份、关系张力、阻力来源、反差点（人前 X / 人后 Y）。
6. **结局倾向**：HE、BE、开放式，但默认 HE。

注意：**题材和钩子手法是两层**。古言 + 替嫁、现言 + 重生、仙侠 + 破镜重圆都是合法组合。不要把它们当互斥项。

如果用户只说"写个女频小说"，默认采用：

```markdown
- 题材：现言
- 钩子手法：追妻火葬场 + 双重背叛 + 绝地反击
- 篇幅：约 3 万字，12-16 章
- 单章：2000-3000 字
- 免费章：前 4 章
- 第一付费节点：第 4 章末
- 关系：撞破/确认伤害 → 被进一步打压 → 女主暗中留证 → 真相裂口 + 反击种子 → 男主追悔付代价 → 女主选择
- 结局：HE
```

## 第三步：短中篇大纲

大纲保存到 `output/<小说名>/大纲.md`。

短中篇大纲必须控制范围，使用**四段式 + 付费节点**：

| 阶段 | 篇幅占比 | 作用 | 必含元素 |
| --- | --- | --- | --- |
| 开局钩子 | 25% | 三秒钩住、前 300 字代入、女主困境、第一轮连环锤 | 第1章强冲突；第2-3章小反抗；第3-5章第一付费节点 |
| 反击蓄能 | 25% | 女主留证、铺退路、关系压迫升级 | 每章 1 个小爽点；不能连续 2 章只虐 |
| 转折爆发 | 30% | 秘密揭露、公开打脸、关系破裂或离开决定 | 1-2 个大爽点（打脸 / 反转 / 身份揭露） |
| 结局收束 | 20% | 清算代价、关系落点、女主重新选择 | 必有 1 个跪式追妻、决绝离场或新生活选择场景 |

**付费节点规则**：

- 第 1 付费节点：第 3-6 章之间，需要"读者必须付费才能知道下一步发生什么"的强卡点
- 后续每 5-7 章设置一个强卡点（不一定是付费墙，但要让读者持续付费阅读）
- 第一付费节点必须卡在"高潮前一秒"：真相/证据/反击种子已出现，行动还没开始
- 付费章下一章必须兑现承诺爽感，不能继续拖
- **最后 5 章不能再开新副线**，必须全程收束

| 要素 | 要求 |

## 第六步：女频投流核心读点

优先写人物选择和情绪因果，但**爽点和卡点的密度高于一切**。详细读点手册见 `references/female-readership.md`。

### 核心要求

1. 女主不是剧情奖品，必须有目标、判断和行动，但必须有让读者代入的"普通起点"。
2. 男主不能只靠身份压迫制造吸引力，必须配齐**三件套**：专属感 / 反差点 / 暗恋证据链。
3. 误会要有信息差来源，不能靠角色强行不说话。
4. 甜和虐都要服务人物关系，但甜虐节奏要交替，**不能连续 3 章只甜或只虐**。
5. 反派和阻力要推动主线，但允许出现"专门给读者打脸代偿用"的工具反派。
6. 结局必须回收核心情感问题、女主成长问题，**且必须回收第1章的钩子和首批金句**。

### 爽点类型（必须循环出现）

| 类型 | 落点 |
| --- | --- |
| 打脸 | 看低女主的人当场被结果反扑 |
| 反转 | 身份 / 关系 / 立场的真相揭露 |
| 占有欲外显 | 男主在公开场合明确"她是我的" |
| 跪舔追妻 | 男主放下身份姿态主动靠近 |
| 装柔实强 | 女主表面退让实则掌控全局 |
| 旧账清算 | 过去的伤害被一次性还回去 |
| 公开偏爱 | 男主在众人面前为女主破例 |
| 身份反转 | 女主真实身份 / 实力被揭露 |

### 章末卡点类型（必须循环出现）

| 类型 | 落点 |
| --- | --- |
| 人物登场 | 关键角色突然出现 |
| 称呼变化 | 一个旧称呼或新称呼炸出旧情 |
| 信息炸弹 | 一句话颠覆前文认知 |
| 动作未完 | 身体语言悬停在某个动作上 |
| 反转预告 | 暗示女主以为 A，下章读者会知道是 B |
| 误会触发 | 一个看似坐实误会的画面 |
| 旧物出现 | 一个有历史的物件浮出 |
| 决定瞬间 | 女主或男主做出关键决定的那一秒 |

详细套路库见 `references/hooks-and-pacing.md`。

## 第七步：质量检查

每章生成后自检。关键项失败时，先修正本章，再进入下一章。

### 章节检查

- [ ] 字数在 2000-3000 字；只有用户明确要求短章才允许 1200-1600 字。
- [ ] 第一句话制造了信息缺口。
- [ ] 前 300 字完成了人设 + 冲突 + 受害/施害关系。
- [ ] 本章爽点落地，不是抽象情绪。
- [ ] 本章锤点清晰，且没有连续 2 章只虐不反抗。
- [ ] 本章事件不是可删的流水账。
- [ ] 女主有主动选择或清晰反应。
- [ ] 男主三锚点至少触发了 1 个。
- [ ] 情绪变化能从前文推导出来。
- [ ] 关系推进没有跳步。
- [ ] 章末卡点强度足够（前 6 章必须达到"信息炸弹"或"反转预告"级别）。
- [ ] 至少产出 1 句金句候选。
- [ ] 有可切片名场面，尤其前 6 章。
- [ ] 没有违反 `STORY_BIBLE.md`。
- [ ] 没有忘记上一章钩子。
- [ ] 同步更新了 `STATE.md` / `PUNCHES.md` / `HOOKS.md` / `GOLDEN-LINES.md`。

### 全篇检查

- [ ] 章节数能支撑 2 万 - 5 万字，不超长膨胀。
- [ ] 前 6 章爽点密度 ≥ 每章 1 个。
- [ ] 前 3 章至少有 1 个小反抗或证据留存。
- [ ] 第一付费节点的卡点足够强。
- [ ] 第一付费节点卡在高潮前一秒，而不是真相全部揭晓之后。
- [ ] 主冲突在中后段开始收束。
- [ ] 伏笔数量可控，临近结局必须回收。
- [ ] 结局解决外部问题、关系问题和女主成长问题。
- [ ] 结局回收了第 1 章的钩子和首批金句。
- [ ] 爽点池至少消费 80%，未消费的不影响主线。

## 第八步：回写第 1 章

短中篇的第 1 章必须重写 3 次。这是硬性流程。

| 触发点 | 回写目的 |
| --- | --- |
| 大纲完成后 | 第1章三重钩要对应大纲的核心冲突 |
| 第10章完成后 | 第1章铺设的伏笔和锚点要和已写章节匹配 |
| 结局完成后 | 第1章必须埋下结局的呼应（金句、动作、物件） |

每次回写必须在 `ERRORS.md` 记录一条"第1章回写日志"，写明改了什么、为什么改。

质量不达标或连续性出错时，记录到 `output/<小说名>/ERRORS.md`。

```markdown
**章节**:
**严重程度**: low | medium | high | critical
**状态**: pending | resolved

### 问题

### 影响

### 修正

### 避免再次发生
```

常见失败（附正反例）：

| 类型 | 反例 | 正例 |
| --- | --- | --- |
| 开篇慢热 | "雨很大，城市像一场梦。" | "我替丈夫养了三年的猫，今天才知道猫主人是他的初恋。" |
| 第一章无缺口 | 用 500 字写女主多美多孤独 | 第一段就让女主撞见背叛、羞辱或反常任务 |
| 只虐不爽 | 连续三章女主被压制，没有任何反击 | 第二章偷偷留证，第三章用证据打回去 |
| 付费点过晚 | 真相全讲完再让读者付费 | 女主看到证据第一帧，正要点开完整视频时卡住 |
| 关系跳步 | 上一章还冷脸分手，这章莫名抱在一起 | 中间必有一个具体事件（陪护一晚、共同危机、旧物触发） |
| 女主失能 | 关键选择都由男主或外力完成 | 主动选择必须由女主触发，男主只能助攻 |
| 误会降智 | 看见男主和女配抱在一起，不问一句就分手 | 看见后冷静离场，第二天派律师送协议，男主才意识到她要走 |
| 情绪断裂 | 上一章还在哭，这章突然嘻嘻哈哈 | 中间必有情绪过渡场景（独处、回忆、关键对话） |
| 篇幅失控 | 中段反复引入新角色和新支线 | 中段所有新元素必须服务主线，无关支线砍掉 |
| 结局仓促 | 最后一章硬解释所有事 | 最后一章只做情绪落点，外部问题在倒数第 3 章前已基本解决 |
| 爽点抽象 | "女主感到很爽" | 具体落到一个动作 / 一句话 / 一个画面 |
| 卡点疲软 | 章末"她回家睡了一觉" | 章末："他从背后叫住她，喊出了五年没人叫过的旧名字。" |

## 图解规则

默认使用 Mermaid，不主动调用图像生成工具。只有用户明确要求生成图片资产时，才使用图像生成。

适合图解的内容：

- 人物关系图
- 情绪曲线
- 爽点 / 卡点 / 金句分布图
- 章节结构图

## 禁止事项

- 不要默认写成男频升级流。
- 不要使用无限升级、无限地图、无限反派递进。
- 不要让女主成为被拯救的道具。
- 不要为了虐而虐：每个虐点必须服务关系或人物。
- 不要靠角色降智制造冲突。
- 不要写"慢热文艺短篇"：投流付费场景下，前 6 章必须有爽点和卡点。
- 不要用纯文学氛围句开篇，除非同一句同时制造信息缺口。
- 不要连续两章只有受虐，没有小反抗、留证、退路或旁人替读者发声。
- 不要把付费点放在真相完全揭晓之后；那是满足点，不是购买点。
- 不要把不同小说的状态混写到同一个共享状态目录。
- 不要跳过大纲直接写正文，除非用户明确要求即兴创作。
- 不要在最后 5 章引入新副线、新反派、新身份反转。
- 不要让章末"什么都没发生"：每个章末都必须卡点。
