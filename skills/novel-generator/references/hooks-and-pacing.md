# 章末卡点与节奏工程手册

投流付费场景下，**章末卡点决定付费转化率**，节奏决定弃读率。这一份是写作时的工程查表，不是理论。

## 一、三秒开篇铁律

投流开篇不是展示文采，是抢注意力。第一句话必须制造信息缺口。

### 第一句公式

```text
反常情境 + 被打破的预期 = 读者问"为什么/然后呢"
```

示例：

```text
闲来无事接了个喂猫的单子，地址竟是闲置两年的婚房。
```

合格原因：

- 反常情境：闲置两年的婚房里为什么有猫。
- 被打破的预期：普通喂猫单变成婚姻悬疑。
- 信息缺口：婚房为什么闲置，猫是谁的，女主和婚房有什么关系。

第一句话不允许只写天气、城市、孤独、漂亮、回忆。除非同一句同时制造信息缺口。

## 二、第一章前 300 字验收

前 300 字必须完成"人设 + 冲突"植入，不要堆设定。

必须回答：

1. 谁在受苦。
2. 谁在施害。
3. 她为什么被这样对待。
4. 读者为什么应该关心。
5. 画面能不能直接当短视频素材。

合格结构：

```text
一个具体场景
  -> 女主被轻视/背叛/羞辱/压迫
  -> 施害者通过动作或一句话出现
  -> 女主露出隐藏优势或反抗苗头
  -> 留下一个必须继续看的问题
```

## 三、第一章前 500 字开篇模板

第一章前 500 字必须见钩。读者看不到钩子，3 秒退出。

### 三种通用开篇钩子

#### 1. 当下危机开局

直接把女主扔进一个具体的、可视化的危机现场。

模板：

```text
[时间/天气一句话] + [女主正在做一件普通事] +
[突然出现的具体冲突] + [一个让读者必须知道答案的悬念]
```

示例：

```text
雨下到傍晚还没停。
沈知意把辞职信压在办公桌左上角。手机响起来，主任的声音不太对：
"最后一桩案子，做完再走。委托人点名要你。对方请的，是盛景的人。"
她拉拉链的手顿住——五年了，第一次有人敢在她面前提"盛景"。
```

关键：必须有一个具体的名字 / 地点 / 旧事被点出来，触发读者想看下一段。

#### 2. 关系冲击开局

第一段就把核心关系角色推到女主面前。

模板：

```text
[女主以为今天是普通的一天] + [她做了一件常规的事] +
[转身/抬头/开门] + [那个人/那件事就在那里]
```

示例：

```text
顾清霜走出民政局的时候，原本以为这一天就这样结束了。
离婚证收进包里，下一站是机场。
出租车停下来，后座的男人摘了墨镜，慢慢看过来。
是顾家三爷。她原本嫁的，是顾家二爷。
```

关键：第一句必须让读者以为日常，最后一句必须让读者意识到一切都不对。

#### 3. 反差身份开局

第一句话就抛出强反差。

模板：

```text
[别人对她的低估] + [她平静接受] + [一个动作泄露真相] + [读者察觉她不是表面那样]
```

示例：

```text
新人培训第三天，主管又把最难的客户甩给她。
"林小满，你来吧，反正你不忙。"
林小满点点头，转身去会议室之前，把别人桌上那份"无解"的并购草案扫了一眼。
五秒之后，她在自己电脑上敲了一行字给老板：
"原方案有问题，下周一前可改完。"
她忘了——她不是新人。她是来收购整家公司的。
```

关键：最后一句必须是身份反转。

## 四、章末卡点套路库

### 套路 1：人物登场

关键角色突然在章末出现。

落点示范：

- "她推开会议室门，里面坐着的人，是她以为再也不会见到的陆沉舟。"
- "电梯门打开，对面站着的——是她明天就要嫁的男人的哥哥。"

适用：第 1 章、关系反转章、第 1 付费节点章。

### 套路 2：称呼变化

一个旧称呼或新称呼炸出旧情。

落点示范：

- "他看着她，先开了口。'知知。'她已经很多年没听过这个称呼。"
- "他从来叫她'裴小姐'。这一次，他说：'阿宁。'"

适用：重逢章、关系转折章。

### 套路 3：信息炸弹

一句话颠覆前文认知。

落点示范：

- "他放下文件，淡淡道：'那个孩子，是我的。'"
- "她翻开档案的最后一页，赫然写着——主刀医生：陆沉舟。"

适用：反转章、揭秘章。

### 套路 4：动作未完

身体语言悬停在某个动作上，章节就此结束。

落点示范：

- "他抬手，似乎想去拨开她耳边的发丝——指尖距她皮肤只剩一厘米——又停住了。"
- "她按下发送键之前，电话先响了。屏幕上跳出的名字，是她删了三年的那个人。"

适用：日常章、暧昧推进章。

### 套路 5：反转预告

明示女主以为 A，下一章读者会知道是 B。

落点示范：

- "她以为他只是顺路。直到第二天，她才看到他名片上的地址——和她家在同一栋。"
- "她笑了笑，关上门。她不知道——门外那个人，刚刚从她父亲案子的复审名单上把自己签了进去。"

适用：中段悬念章。

### 套路 6：误会触发

一个看似坐实误会的画面。

落点示范：

- "她在走廊上转过那个弯——他正抱着另一个女人，那女人正在哭。"
- "电梯门合上的最后一秒，她看清了——他西装上的口红印。"

适用：拉扯章、虐点章。

注意：误会必须在 3 章内推进，不能拖。

### 套路 7：旧物出现

一个有历史的物件浮出，作为暗恋证据或伤痛触发器。

落点示范：

- "他从抽屉最深处拿出来一样东西，放在桌上。那是她大学时丢的那本《查令十字街84号》，扉页上是她自己的字。"
- "她从他书架上抽下一本书，一片干枯的桂花从里面落下来——那是她十年前夹进去的。"

适用：暗恋揭露章、和解章。

### 套路 8：决定瞬间

女主或男主做出关键决定的那一秒。

落点示范：

- "她在合同最后一页签下名字。从今天起，她叫顾清霜。"
- "他拿起手机：'订最近一班去 S 市的票。'秘书愣住——那是他十年不肯踏足的城市。"

适用：阶段转折章。

## 五、卡点强度分级

不是所有章末都需要同等强度。强度按章节位置分配。

| 章节位置 | 卡点强度要求 | 推荐套路 |
| --- | --- | --- |
| 第 1 章 | 极强 | 人物登场 / 称呼变化 / 信息炸弹 |
| 免费章末（第 3-6 章） | 极强（第一付费节点） | 信息炸弹 / 反转预告 / 决定瞬间 |
| 关系升温段 | 中-强 | 动作未完 / 称呼变化 / 旧物出现 |
| 大爽点章 | 强 | 信息炸弹 / 决定瞬间 |
| 火葬场段 | 强 | 决定瞬间 / 误会触发 |
| 结局前一章 | 极强（最后悬念） | 反转预告 / 信息炸弹 |
| 结局章 | 不卡，落点 | 旧物出现 / 称呼变化（首尾呼应） |

铁律：**前 6 章每章卡点必须达到"极强"或"强"，不允许出现"中"以下的卡点**。

## 六、爽点密度规则

爽点是按密度分配的，不是按章节平均分配。

### 全篇爽点配比（按 20 章为例）

| 爽点级别 | 数量 | 位置 |
| --- | --- | --- |
| 小爽点 | 12-15 个 | 几乎每章 1 个 |
| 中爽点 | 4-6 个 | 每 3-4 章 1 个 |
| 大爽点 | 2-3 个 | 第 1 付费节点 / 中段反转 / 结局前 |

### 小爽点示例

- 男主一个不经意的专属动作（章中触发，章末再次复现）
- 女主一次小布局成功（让读者意识到她不是任人摆布）
- 一个反派被女主一句话怼回去
- 一次"路人甲注意到男主对女主不一样"的旁观视角

### 中爽点示例

- 一次完整的小型打脸（解决一个反派的当前威胁）
- 一次关系突破（从对立到第一次合作）
- 一次身份小揭露（女主或男主某个细节被部分揭开）

### 大爽点示例

- 完整的身份反转 / 旧账清算
- 公开占有欲宣示场景
- 男主跪式追妻的关键节点

## 七、连环锤情绪节奏

投流读者不是不能接受虐，而是不能接受"一直惨且没有希望"。每一轮情绪锤必须递进。

### 基本链条

```text
施压/受虐 -> 女主反应 -> 小反抗或留证 -> 更高一级冲突 -> 章末卡点
```

可用节奏：

```text
虐一次 + 小反抗 + 再虐 + 升级反抗
```

禁止节奏：

```text
连虐三章 + 女主无行动 + 只靠男主或外力救场
```

### 小爽点不等于大反击

三章内必须给读者一个"她不会一直惨"的信号：

- 证据被悄悄保存。
- 退路被暗中铺好。
- 旁人发来一句"小心"。
- 女主当场不爆发，但回头发出律师函、录音、截图、协议。

## 八、甜虐比例落地表

提示词中的"甜虐比例"必须落到具体章节。

### 6 甜 4 虐（推荐默认）

按 20 章计：

- 第 1-3 章：拉扯（甜中带涩）
- 第 4-6 章：第一付费节点 + 短虐 + 短甜（建立关系吸引）
- 第 7-10 章：甜（关系升温，铺暗恋证据）
- 第 11-13 章：大虐（真相揭露 / 关系破裂）
- 第 14-17 章：火葬场（男主单方面甜 + 女主继续虐）
- 第 18-20 章：和解甜 + 结局回收

### 4 甜 6 虐（虐恋偏向）

- 前期短甜建立信任
- 中段双虐
- 结局段才出现真正的甜

### 8 甜 2 虐（轻松向）

- 全程甜，但每 3-4 章插入一个短虐用作章末卡点
- 不能出现连续 3 章无虐——读者会麻木

铁律：**不允许连续 3 章只甜或只虐**。情绪必须交替。

## 九、付费卡点专项设计

第 1 付费节点（一般第 3-6 章末）的卡点设计独立于普通卡点，必须满足：

1. **不付费就无法理解的信息缺口**：读者必须强烈想知道"下一章会发生什么"。
2. **情绪强度峰值**：本章末的情绪冲击力必须是免费章中最高的。
3. **角色行动悬置**：关键人物刚刚做了一个动作，停在半空。
4. **承诺爽感**：让读者相信"付费后会立刻给到一个大爽点"，且付费章必须真的给到。

核心判断：付费点不是高潮，而是高潮前的那一秒。

合格状态：

```text
情绪蓄能已完成
  -> 真相/证据/反击种子刚露出
  -> 女主即将行动
  -> 章节结束
```

不合格状态：

```text
全部真相已经讲完
  -> 女主已经反击成功
  -> 读者情绪已经满足
  -> 再要求付费
```

### 付费卡点示范

```text
（章节末尾）
许老太把鉴定书推到她面前。
最后一页签字的位置，赫然写着两个名字。
一个是她父亲——五年前已经过世。
另一个，是陆沉舟。
她抬起头：
"为什么……是你？"
他没有回答，只把袖口慢慢挽起来，露出腕骨那道她以为只是疤的旧伤。
那不是疤。是她父亲案卷上提到过的——那一夜真正在场的人，唯一的证据。
```

要点：

- 信息缺口（父亲案子真相）
- 情绪峰值（女主认知崩塌）
- 角色行动悬置（男主挽袖、没回答）
- 承诺爽感（下章必定揭开真相）

## 十、节奏雷区

下列结构必删：

| 雷区 | 描述 | 修正 |
| --- | --- | --- |
| 第一句无缺口 | 只写天气、城市、孤独、漂亮 | 第一行就给反常情境和被打破的预期 |
| 前 300 字无冲突 | 只铺人设和氛围 | 直接写女主被背叛、被轻视、被羞辱或被迫做选择 |
| 慢热三章 | 前 3 章只在建立背景 | 第 1 章必须有冲击事件 |
| 连虐无反抗 | 女主连续挨打，没有留证或退路 | 三章内安排小爽点，证明她不会一直惨 |
| 章末降温 | 章末用平静场景收尾 | 任何章末都必须有卡点 |
| 内心戏堆积 | 整章 80% 是心理描写 | 心理描写不能超过 30% |
| 信息倾倒 | 一章把所有背景交代完 | 背景拆成 5-8 个章节碎片 |
| 误会拖延 | 一句话能解释的误会拖 10 章 | 主线误会最多拖 3-5 章 |
| 反派抢戏 | 反派的戏份多过男主 | 反派只服务打脸场景 |
| 付费点放错 | 真相全揭晓、情绪已满足后才卡 | 卡在高潮前一秒，行动留给付费章 |
| 结局赶工 | 最后 2 章解决所有事 | 倒数第 3 章前主线已基本完成 |

## 十一、回写第 1 章 checklist

每次回写第 1 章必须确认：

### 大纲完成后

- [ ] 第 1 章的"3 重钩"（冲突钩 / 关系钩 / 反差钩）对应大纲核心冲突。
- [ ] 第 1 章铺设的视觉锚点 / 语言锚点出现在大纲至少 3 个章节。
- [ ] 第 1 章末的卡点为大纲第 2 章开篇做了承接。

### 第 10 章完成后

- [ ] 第 1 章铺设的暗恋证据，已在前 10 章触发至少 1 条。
- [ ] 第 1 章的女主反差感，在前 10 章被证实至少 1 次。
- [ ] 第 1 章如果有未来呼应的物件 / 称呼，已规划好在哪一章出现。

### 结局完成后

- [ ] 第 1 章的金句在结局章被复现 / 反转。
- [ ] 第 1 章末的卡点（人物 / 称呼 / 物件）在结局回应。
- [ ] 第 1 章女主的初始状态 vs 结局状态形成清晰对照。

每次回写必须在 ERRORS.md 记录"第 1 章回写日志"。
