# 女频短中篇结构参考

用于 2w-5w 字、默认每章 2000-3000 字的女频投流付费短中篇。用户明确要求短章时，才使用 1200-1600 字。

## 四段式 + 付费节点

```text
开局钩子（15%）
  -> 第1章 3重钩（冲突钩 / 关系钩 / 反差钩）
  -> 前 6 章建立女主反差感、男主专属感
  -> 第 1 付费节点必须在第 3-6 章末

关系升温（35%）
  -> 每章 1 个小爽点
  -> 每 3-4 章 1 个中爽点
  -> 暗恋证据链 / 男主三件套 持续触发

转折爆发（30%）
  -> 中段 1 个大反转或大爽点
  -> 主冲突推到峰值
  -> 女主关键选择（不是被动接受）

结局收束（20%）
  -> 65%-85% 火葬场段（男主代价具体）
  -> 最后 5 章不开新副线
  -> 结局回收第 1 章钩子、金句、物件
```

## 投流爆款三类模板

### 模板一：双重背叛 + 绝地反击

适合现言、婚恋、家庭冷暴力、出轨、豪门压迫。

```text
第 1 章：撞破背叛或确认羞辱，开篇直接给情绪冲击
第 2-3 章：被进一步打压，但女主暗中留证或铺退路
第 4-5 章：亮出第一张底牌，第一付费节点卡在反击前一秒
第 6-8 章：层层打脸，爽点密集兑现
后段：旧账清算、男主/反派付代价、女主重新选择
```

关键要求：

- 第 1 章不能只怀疑，必须让读者看到具体伤害。
- 第 3 章前必须出现小反抗，不能让女主一直惨。
- 第一付费节点必须是"她终于有证据，但还没动手"。

### 模板二：重生觉醒 + 重新选择

适合复仇爽文、婚恋错付、家族吸血、前世惨死。

```text
第 1 章：前世惨死 / 被辜负的一生 + 重生回关键节点
第 2 章：做出和前世不同的选择
第 3-4 章：退婚 / 分手 / 离开，展露新姿态
第 5-6 章：渣男/渣女不适应，开始后悔，第一付费节点
第 7-8 章：新生活 + 最终打脸的第一轮兑现
后段：系统性清算，女主拿回人生
```

关键要求：

- 重生不是回忆工具，第一章必须立刻改变一个结果。
- 女主不能只靠前世信息开挂，必须有行动和判断。
- 每个清算节点都要对应一个明确爽点。

### 模板三：替身觉醒 + 身份反转

适合替身、契约婚姻、白月光回归、工具人觉醒。

```text
第 1 章：发现自己是替身 / 工具人，直接给羞辱名场面
第 2-3 章：进一步被羞辱，女主决定离开并准备退路
第 4-5 章：离开后的艰难 + 新机遇，第一付费节点
第 6-7 章：渣男发现真相，追悔莫及
第 8 章：最终选择，新生活或渣男下跪
后段：身份反转、公开打脸、关系落点
```

关键要求：

- 替身身份必须早暴露，不能藏到中段。
- 女主离开必须有具体代价，不能像换场景一样轻。
- 男主后悔要付实际代价，不靠嘴。

## 章节规模建议

| 总字数 | 单章字数 | 建议章节 | 免费章 | 第一付费节点 |
| --- | --- | --- | --- | --- |
| 2 万 | 2000-2500 | 8-10 章 | 前 3 章 | 第 3 章末 |
| 3 万 | 2000-3000 | 10-15 章 | 前 3-4 章 | 第 4 章末 |
| 5 万 | 2500-3000 | 16-20 章 | 前 4-5 章 | 第 5 章末 |

用户明确要求短章时，才使用 1200-1600 字并增加章节数。

## 爽点密度规则（按 20 章为例）

| 爽点级别 | 数量 | 位置 |
| --- | --- | --- |
| 小爽点 | 12-15 个 | 几乎每章 1 个 |
| 中爽点 | 4-6 个 | 每 3-4 章 1 个 |
| 大爽点 | 2-3 个 | 第 1 付费节点 / 中段反转 / 结局前 |

详细爽点类型见 `female-readership.md`。

## 章末卡点强度分级

| 章节位置 | 卡点强度 |
| --- | --- |
| 第 1 章 | 极强 |
| 免费章末（含第一付费节点） | 极强 |
| 关系升温段 | 中-强 |
| 大爽点章、火葬场段 | 强 |
| 结局前一章 | 极强 |
| 结局章 | 不卡，做落点和回收 |

详细卡点套路见 `hooks-and-pacing.md`。

## 典型钩子手法结构

### 破镜重圆 + 追妻火葬场

```text
旧伤重逢 -> 被迫合作 -> 旧误会浮出 -> 短暂靠近 ->
真相爆发 -> 女主决绝离场 -> 男主火葬场追妻 -> 重新建立关系
```

关键点：

- 分手原因必须具体，且与男主当年的"无法解释"有关（保护 / 替担 / 家族压力）。
- 复合不能只靠怀旧。男主必须付出可见代价。
- 火葬场段占全文 20% 左右，不能太短。

### 先婚后爱（替嫁 / 契约）

```text
利益绑定 -> 规则约定 -> 日常试探 -> 互相看见真实处境 ->
外部压力 -> 主动选择彼此 -> 危机考验 -> 共同承担
```

关键点：

- 婚姻不是终点，是压力容器。
- 关系升温来自共同经历。
- 结局要回答"为什么现在是真爱"。
- 必须设计至少一次"对方差点失去自己"的危机。

### 暗恋成真

```text
单向隐藏 -> 意外接近 -> 误读与期待 -> 秘密暴露 ->
对方回应 -> 双向确认 -> 共同面对外部阻力
```

关键点：

- 暗恋者不能一直被动等待。
- 对方被吸引需要具体原因，最好早期就埋暗恋证据链（实际上是双向暗恋）。
- 暗恋曝光必须带来代价或选择。

### 重生归来（清算式）

```text
重生第一击（小打脸） -> 摸清前世真相 -> 系统性布局 ->
逐个清算 -> 与男主关系重写 -> 最终大清算 -> 新生
```

关键点：

- 重生的第 1 章必须立刻出现"前世悲剧"画面 + 重生的爽感。
- 女主必须主动改命，不能只复述前世信息。
- 每个清算节点对应一个爽点。
- 与男主的关系必须有"前世亏欠 / 误解"做戏剧动力。

### 马甲文（隐藏身份）

```text
低姿态登场 -> 被低估 -> 小马甲掉一片 ->
读者得意 -> 主角周围人疑惑 -> 大马甲翻车 -> 全面身份揭露
```

关键点：

- 马甲必须在前 3 章就埋好 2-3 张。
- 每章可以掉一小片马甲（小爽点），积累到关键章节翻大马甲。
- 男主可以早一步知道，但保留秘密——这本身就是占有欲表达。

### 强强

```text
立场冲突 -> 能力交锋 -> 互相认可 ->
利益分歧 -> 并肩承担 -> 平等选择
```

关键点：

- 双方都有能力和目标。
- 爱情不能削弱女主。
- 冲突来自立场，不是无意义吵架。
- 必须有"对方欣赏女主能力"的高光时刻（公开认可）。

## 情绪曲线

### 酸涩治愈

```text
克制疏离 -> 被迫接近 -> 微弱心动 -> 旧伤刺痛 -> 坦白与理解 -> 温柔落点
```

### 甜虐拉扯

```text
吸引 -> 误会 -> 靠近 -> 伤害 -> 失控 -> 选择 -> 和解
```

### 事业爱情双线

```text
事业困境 -> 关系干扰 -> 双线互相施压 -> 事业突破 -> 情感选择 -> 双线收束
```

### 复仇逆袭

```text
被压制 -> 隐忍布局 -> 第一次反击 -> 真相揭露 ->
全面反扑 -> 男主助攻或并肩 -> 清算完成 -> 关系落点
```

## 伏笔控制

短中篇不要埋太多伏笔。

| 伏笔类型 | 回收间隔 |
| --- | --- |
| 短伏笔 | 1-3 章 |
| 中伏笔 | 4-8 章 |
| 核心秘密 | 中后段必须揭露 |

规则：

- 同时存在的待回收伏笔不超过 5 个。
- 进入最后 20% 篇幅后，不再新增核心伏笔。
- 结局前必须回收主冲突相关伏笔。
- 第 1 章铺设的物件 / 称呼 / 金句必须在结局回应。

## 差结构信号

- 第一句话只有文采，没有信息缺口。
- 前 300 字看不出谁受苦、谁施害、为什么要关心。
- 第 1 章前 500 字没出现钩子。
- 第 1 章末不是"极强"卡点。
- 前 3 章没有小爽点、留证、退路或反抗苗头。
- 第 5 章还没看出核心关系。
- 第一付费节点放在真相全部揭晓之后。
- 中段一直新增角色和支线。
- 女主的目标被男主目标吞掉。
- 误会靠"不解释"硬拖超过 3 章。
- 最后三章才突然解决所有问题。
- 章末"什么都没发生"。
- 连续 3 章只有甜或只有虐。
- 前 6 章没有任何身体吸引力的建立（缺少 L1-L2 级暧昧场景）。
- 暧昧场景跳级（前 3 章出现 L4）。
- 暧昧场景没有叙事功能，为擦而擦。
